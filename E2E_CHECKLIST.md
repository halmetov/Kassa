# E2E чек-лист после обновлений

1. **Приход (мобайл <768px)**
   - Открыть вкладку "Приход по накладным", убедиться, что строки отображаются карточками в одну колонку.
   - Добавить несколько позиций, изменить количество/цены, удалить строку — кнопки "Добавить строку" и "Сохранить приход" должны быть заметны и закреплены.
   - Сохранить приход и убедиться, что история обновилась без ошибок.
2. **Склад**
   - Выбрать филиал через фильтр и проверить, что таблица показывает только колонки Товар/Кол-во/Лимит.
3. **Касса (мобайл)**
   - Добавить товары в корзину: нижняя панель не перекрывается системной навигацией, на вкладке "Корзина" отображается красный бейдж с количеством позиций.
   - Переключение вкладок не ломает содержимое.
4. **Товары**
   - В таблице видны только Фото/Название/Кол-во и кнопка просмотра.
   - Открыть модалку "Подробнее": для админа доступны редактирование и удаление с подтверждением, для сотрудника только просмотр.
5. **Отчеты**
   - Таблица показывает номер заказа, сотрудника и сумму; кнопка "Подробнее" открывает модалку с деталями (дата, позиции, оплаты, клиент/долг).
6. **Перемещения**
   - Любой пользователь может выбрать любой филиал "откуда" и "куда", создать перемещение и увидеть его в списке. Проверить корректность остатков и статусов.
7. **Касса (backend)**
   - Создать продажу и убедиться, что она проводится по филиалу "Магазин" вне зависимости от филиала пользователя; остатки уменьшаются в магазине.
